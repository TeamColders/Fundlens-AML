# FundLens — AI-Powered AML Fraud Detection System

## Problem Statement
This project addresses the Union Bank of India's Fund flow / AML problem statement by automating the detection of FATF-recognized typologies using graph analytics, GNN scoring, and Generative AI for bilingual STR drafting.

## Live Demo
Run locally using instructions below.

🔗 Live Demo: https://fundlens-aml-delta.vercel.app/
🎥 Demo Video: https://www.youtube.com/watch?v=clhQhC5F7mk

## Tech Stack
• Python 3.11+
• FastAPI (backend API)
• Neo4j (graph database)
• PostgreSQL / SQLite (relational data & evidence chain)
• Google Gemini (STR generation + NL query)
• React, Vite, Tailwind CSS (frontend UI)
• PyTorch Geometric (GNN model scoring)
• Kafka & Redis (optional ingestion pipeline)

## How to Run Locally
1. Clone the repo:   git clone https://github.com/your-team/Fundlens-AML
2. Navigate to folder: cd Fundlens-AML
3. Setup python environment:   python3 -m venv venv && source venv/bin/activate
4. Install dependencies:   pip install -r requirements.txt && npm install
5. Setup env variables:   cp .env.example .env (and set GEMINI_API_KEY)
6. Generate synthetic data:   python3 backend/database/demo_seed.py --mode local (or ./run_demo.sh)
7. Run the API (Terminal 1):   python3 -m uvicorn backend.api.main:app --reload --port 8000
8. Launch the dashboard (Terminal 2):   npm run dev
9. Open browser at:   http://localhost:5173

## Project Structure
/backend — FastAPI app, database seeds, graph queries, Gemini STR logic, and evidence chain
/src — React + Vite UI application with Investigation Dashboard and network visualizations
/data — Synthetic data generators, demo seeds, and case JSONs
/docker — Docker configuration files for full stack deployment (Neo4j, Postgres, Kafka)
/requirements.txt — core Python dependencies
/requirements-ml.txt — optional ML dependencies for GNN

## Dataset
Demo data is 100% synthetic, generated by our team and seeded into SQLite/Postgres using `demo_seed.py`. It simulates realistic bank transactions, entity profiles, and FATF typologies (e.g., round-trip layering, mule chains) to showcase the graph analytics capabilities.

No real bank data was used.

## Model Performance (on Synthetic Test Set)
GNN Anomaly Detector:
• Precision: 0.869 | Recall: 0.962 | F1: 0.913
• AUC-ROC: 0.993
• False Positive Rate: 4.2%

Note: These results are on synthetic data. Performance on real bank data would require re-training and fine-tuning.

## Known Limitations
• Trained only on synthetic data; would need real transaction data for production use.
• Full real-time ingestion pipeline (Kafka) requires more extensive testing at massive scale.
• Gemini STR drafting requires an active API key and may hit rate limits on free tiers.
• GNN model requires periodic retraining to adapt to shifting fraud typologies.
